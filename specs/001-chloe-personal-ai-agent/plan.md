# Implementation Plan: Chloe ‚Äî Personal AI Agent

**Branch**: `001-chloe-personal-ai-agent` | **Date**: 2026-04-13 | **Spec**: [spec.md](spec.md)

## Summary

Chloe is a personal AI agent built on Bun + TypeScript (strictest settings), powered by Anthropic Claude with streaming, a ReAct tool-use loop, human-in-the-loop confirmation, named sessions persisted in SQLite, and a plugin-ready tool registry. The implementation exposes a single `@chloe/core` library consumed by two entry points: a CLI (interactive REPL) and a REST/SSE API service.

## Technical Context

**Language/Version**: TypeScript 5.x, Bun ‚â• 1.1
**Primary Dependencies**: `@anthropic-ai/sdk`, `bun:sqlite` (built-in), Biome, `bun-types`
**Storage**: SQLite via `bun:sqlite` (built-in, no extra dependency)
**Testing**: `bun test` (built-in)
**Target Platform**: Linux/macOS (personal workstation, WSL2 supported)
**Project Type**: Bun workspace monorepo ‚Äî library + CLI + API service
**Performance Goals**: Real-time token streaming (<50ms first-token latency after model starts responding); CLI startup <200ms
**Constraints**: No network required for storage; no auth on API (personal use); API is single-user
**Scale/Scope**: Single user, personal assistant; session history unbounded for v1

## Implementation Phases

### Phase 1 ‚Äî Project Scaffolding

**Goal**: A compiling, linting, testing skeleton with all packages wired together.

**Tasks**:
1. Create root `package.json` (workspace definition, scripts: `build`, `test`, `check`)
2. Create root `tsconfig.json` (strict settings, project references)
3. Create root `biome.json` (linting + formatting rules)
4. Create `packages/core/package.json`, `packages/core/tsconfig.json`
5. Create `packages/cli/package.json`, `packages/cli/tsconfig.json` (dep: `@chloe/core: workspace:*`)
6. Create `packages/api/package.json`, `packages/api/tsconfig.json` (dep: `@chloe/core: workspace:*`)
7. Add placeholder `src/index.ts` to each package (exports one empty const)
8. Run `bun install`, verify workspace links
9. Run `biome check .` ‚Äî passes with empty files
10. Run `bun test` ‚Äî passes with zero tests

**Done when**: `bun install && biome check . && bun test` all exit 0 from repo root.

### Phase 2 ‚Äî Core: Storage Layer

**Goal**: Persistent SQLite-backed sessions and messages behind the `StorageAdapter` interface.

**Tasks**:
1. Define `StorageAdapter` interface in `packages/core/src/storage/adapter.ts`
2. Define `Session`, `SessionSummary`, `Message` types in `packages/core/src/session/types.ts`
3. Implement `SQLiteStorageAdapter` in `packages/core/src/storage/sqlite.ts`:
   - Constructor: `new Database(path, { create: true })`, apply pragmas, run DDL
   - Implement all `StorageAdapter` methods using prepared statements
4. Implement `slugify()` + `validateSessionId()` in `packages/core/src/session/slug.ts`
5. Write unit tests in `packages/core/src/storage/sqlite.test.ts`:
   - Use in-memory database (`:memory:`)
   - Test: create session, get session, list sessions, delete session (cascade), append messages, get messages
6. Write unit tests in `packages/core/src/session/slug.test.ts`:
   - Test slugification rules (spaces, special chars, empty result rejection, length limit)

**Done when**: All unit tests pass; `biome check` clean.

---

### Phase 3 ‚Äî Core: Tool Registry

**Goal**: A type-safe tool registry with the EchoTool reference implementation.

**Tasks**:
1. Define `Tool` interface in `packages/core/src/tools/types.ts`
2. Implement `ToolRegistry` in `packages/core/src/tools/registry.ts`:
   - `register(tool: Tool): void` ‚Äî add tool, error on duplicate name
   - `get(name: string): Tool | null`
   - `list(): Tool[]` ‚Äî returns all registered tools as Anthropic `Tool[]` shape
3. Implement `EchoTool` in `packages/core/src/tools/echo.ts`:
   - Input schema: `{ message: string }`
   - Execute: returns `message` unchanged as string
4. Write unit tests in `packages/core/src/tools/registry.test.ts`:
   - Test: register + get, get unknown returns null, list returns correct Anthropic shape, duplicate registration error
5. Write unit tests in `packages/core/src/tools/echo.test.ts`:
   - Test: execute returns input message

**Done when**: All tests pass; `biome check` clean.

---

### Phase 4 ‚Äî Core: ReAct Agent Loop

**Goal**: The streaming ReAct loop with human-in-the-loop callback support.

**Tasks**:
1. Define `AgentConfig`, `AgentCallbacks`, `RunResult` types in `packages/core/src/agent/types.ts`:
   - `AgentConfig`: `{ model, apiKey, tools, storage }`
   - `AgentCallbacks`: `{ onToken?, onToolCall?, onToolResult?, confirmTool? }`
     - `confirmTool?: (name: string, input: unknown) => Promise<boolean>`
2. Implement `runLoop()` in `packages/core/src/agent/loop.ts`:
   - Accepts: `{ messages, client, tools, callbacks }`
   - ReAct loop as described in research.md
   - Streams text deltas via `callbacks.onToken`
   - Calls `callbacks.confirmTool` if set; if returns false, injects denial observation
   - Returns final `RunResult` with full updated message history
3. Implement `createAgent()` / `Agent` class in `packages/core/src/agent/agent.ts`:
   - Wraps `runLoop()` with session management (load history ‚Üí run ‚Üí save new messages)
   - `agent.run(sessionId, userMessage, callbacks): Promise<RunResult>`
4. Write unit tests in `packages/core/src/agent/loop.test.ts`:
   - Use mock Anthropic client (stub `messages.stream()`)
   - Test: single turn no tools (end_turn), single tool call confirmed, single tool call denied, multi-turn loop, unknown tool graceful handling
5. Export all public APIs from `packages/core/src/index.ts`

**Done when**: All loop unit tests pass (using mocked Anthropic client); `biome check` clean.

---

### Phase 5 ‚Äî CLI: Chat Command

**Goal**: Interactive streaming REPL with human-in-the-loop confirmation.

**Tasks**:
1. Implement `confirm()` in `packages/cli/src/ui/confirm.ts`:
   - Uses `readline` to prompt `[tool] <name>\n  input: ...\nConfirm? [y/N]: `
   - Returns `Promise<boolean>`
2. Implement streaming printer in `packages/cli/src/ui/stream.ts`:
   - `printToken(text: string): void` ‚Äî writes to stdout without newline
   - `printLine(text: string): void` ‚Äî writes with newline
3. Implement `chatCommand()` in `packages/cli/src/commands/chat.ts`:
   - Args: `--session <name>` (required), `--yes` (optional)
   - Creates/resumes session via `Agent`
   - Runs readline REPL loop
   - `exit` / Ctrl-C: clean shutdown
4. Implement CLI entry `packages/cli/src/index.ts`:
   - `parseArgs()` using `bun`'s `parseArgs` or `process.argv`
   - Dispatch to `chatCommand`, `serveCommand`, `sessionsCommand`
   - Fail fast with helpful error if `ANTHROPIC_API_KEY` is absent

**Done when**: `chloe chat --session test` runs, sends one message, receives a streamed response, and exits cleanly. Tool confirmation prompt works.

---

### Phase 6 ‚Äî CLI: Serve Command + API Handlers

**Goal**: REST/SSE API service matching the contract in `contracts/api.md`.

**Tasks**:
1. Implement `messages` handler in `packages/api/src/handlers/messages.ts`:
   - `POST /sessions/:id/messages` ‚Üí SSE stream
   - Validate request body; return 400 if `content` missing
   - Create `ReadableStream` forwarding `onToken` events + `[DONE]` sentinel
   - Auto-confirm tool calls in API mode (no HITL in API)
2. Implement `sessions` handler in `packages/api/src/handlers/sessions.ts`:
   - `GET /sessions` ‚Üí JSON array of `SessionSummary`
   - `DELETE /sessions/:id` ‚Üí `{ deleted: true }` or 404
3. Implement router in `packages/api/src/router.ts`:
   - Pattern matching via `URL.pathname` + `Request.method`
   - Returns 404 JSON for unknown routes, 405 for wrong method
4. Implement server in `packages/api/src/index.ts`:
   - `Bun.serve()` with port from `--port` / `PORT` env / default 3000
5. Implement `serveCommand()` in `packages/cli/src/commands/serve.ts`:
   - Starts `packages/api` server with given port
   - Prints `Chloe API listening on http://localhost:<port>`

**Done when**: `chloe serve` starts; `curl -N -X POST http://localhost:3000/sessions/test/messages -H 'Content-Type: application/json' -d '{"content":"Hello"}'` returns SSE stream.

**Done when**: `chloe sessions list` shows created sessions; `chloe sessions delete <id>` removes one.
