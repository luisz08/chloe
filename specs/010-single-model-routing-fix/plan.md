# Implementation Plan: Single-Model Routing Fix & Refactor Residue Cleanup

**Branch**: `010-single-model-routing-fix` | **Date**: 2026-04-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/010-single-model-routing-fix/spec.md`

## Summary

Gate subagent delegation tools (`vision_analyze`, `fast_query`, `deep_reasoning`) and their associated system prompt on the effective resolved model configuration: when every specialized model falls back to `default_model`, treat the agent as single-model and skip subagent tool registration plus subagent prompt injection. Also clean up residue left by the route-token → subagent-tool refactor (dead types, dead parameters, legacy parallel loop, stale test comments, and a stale spec 009).

Technical approach: derive a `multiModel` boolean in the `Agent` constructor from the resolved `ResolvedModelConfig` (true if any of `reasoning`, `fast`, or `vision` differs from `default`), and use it as a single guard for both tool registration and system-prompt inclusion. Propagate the guard through a new optional `system` field on the loop options type, so the loop itself has no knowledge of subagent semantics. Collapse `runLoop` and `routingRunLoop` into one implementation, delete dead types, and mark `specs/009-multi-model-routing/` documents as superseded.
