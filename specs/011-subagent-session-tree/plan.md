# Implementation Plan: Subagent Session Tree

**Branch**: `011-subagent-session-tree` | **Date**: 2026-04-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/011-subagent-session-tree/spec.md`

## Summary

Track subagent calls (vision_analyze, fast_query, deep_reasoning) as child sessions with parent association. Each subagent invocation creates a child session record, persists full request/response messages with metadata (tokens, model, elapsed time), and returns only text to parent agent. CLI and API provide tree view and children query interfaces.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode: `strict: true`, `exactOptionalPropertyTypes: true`, `noUncheckedIndexedAccess: true`)
**Primary Dependencies**: Bun ≥ 1.1, @anthropic-ai/sdk, bun:sqlite
**Storage**: SQLite via `bun:sqlite` (SQLiteStorageAdapter)
**Testing**: `bun test`
**Target Platform**: Linux/macOS CLI and HTTP API server
**Project Type**: Monorepo CLI + API (packages: core, cli, api)
**Performance Goals**: Session tree query < 1s for 50 children, single call history < 2s
**Constraints**: Parent agent behavior unchanged (API compatibility), no automatic cleanup
**Scale/Scope**: Up to 10 nesting levels, timestamp precision milliseconds

## Phase 0: Research Summary

See [research.md](./research.md) for detailed findings.

Key decisions:
1. **Session ID format**: `{parentId}-{type}-{timestamp}` — deterministic, sortable
2. **Metadata storage**: JSON in existing `content` field via structured metadata object
3. **Tree query**: Recursive CTE in SQLite for efficient tree traversal
4. **Failure handling**: Create session before API call, update on error
