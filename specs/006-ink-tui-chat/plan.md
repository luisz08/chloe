# Implementation Plan: ink TUI Chat Interface

**Branch**: `006-ink-tui-chat` | **Date**: 2026-04-15 | **Spec**: specs/006-ink-tui-chat/spec.md

## Summary

Replace `chloe chat`'s `readline`-based interface with a full-screen ink TUI. The UI renders role-differentiated message bubbles (You / Chloe / Tool), streams assistant replies token-by-token with a live cursor, renders Markdown via `Bun.markdown.ansi()`, confirms tool calls inline, and shows a fixed bottom status bar. One additive change to `@chloe/core` is required: adding an optional `onUsage` callback to `AgentCallbacks`.

## Key Implementation Notes

### Avoiding the readline/ink conflict

**Critical**: Never create a `readline.Interface` in the same process as an ink render. The current `chat.ts` uses readline — it must be completely replaced. Remove `createInterface` imports. All input goes through ink's `useInput`.

### Streaming token debounce

Tokens arrive faster than React renders. Use a `useRef` to accumulate content without triggering state updates on every token, then flush to state every ~16ms:

```ts
const buffer = useRef("");
const flush = useCallback(() => {
  if (buffer.current !== messages[lastIdx].content) {
    setMessages(prev => /* update last message content from buffer */);
  }
}, [...]);
useEffect(() => {
  const id = setInterval(flush, 16);
  return () => clearInterval(id);
}, [flush]);
```

On `onToken`, append to `buffer.current` only. This keeps renders at ~60fps regardless of token rate.

### Tool confirmation in the App loop

`confirmTool` in `AgentCallbacks` must return a `Promise<boolean>`. In the ink context, resolve this promise when the user presses `y` or `N` in the `ToolBlock`:

```ts
confirmTool: async (name, input) => {
  return new Promise<boolean>((resolve) => {
    // Add a pending ToolBlock to messages
    // Store resolve in a ref; ToolBlock calls resolve(true/false) on keypress
  });
}
```

### Bun.markdown.ansi() usage

```ts
import { markdown } from "bun";  // or Bun.markdown.ansi() directly

const rendered = Bun.markdown.ansi(content, {
  colors: true,
  columns: process.stdout.columns ?? 80,
});
```

Pass `rendered` directly as a string child of ink's `<Text>` — ANSI codes are preserved.

### ink render options

```ts
import { render } from "ink";
render(<App ... />, {
  exitOnCtrlC: false,   // we handle Ctrl+C ourselves (double-press)
});
```
